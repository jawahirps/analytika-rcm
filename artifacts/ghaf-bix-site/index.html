Why Ghaf BIX

RCM teams lose revenue in the gap between portals and spreadsheets.

Claims live in the DHA portal. Remittances arrive as XML. Reconciliation happens in Excel — late, manual, and per-facility. Ghaf BIX closes that gap with one automated pipeline from portal to report.

🔄

Autonomous Portal Sync

Credential-managed DHA & Riayati sync with resumable, stall-hardened downloads. Interrupted at record #12,226? It picks up exactly there — no re-runs, no gaps.

🧩

XML Parsing Workbench

Submission and remittance-advice XML parsed into structured claims, activities and payment rows — with match counters for sub files, RA rows, and open references.

⚖️

Remittance Reconciliation

Automatic matching of remittance lines to submitted claims surfaces underpayments, denials and unmatched references before they age out of the resubmission window.

📊

Claim Summary & Advanced Reports

Scheduled or on-demand reports filtered by facility, payer, clinician, department and encounter dates — generated in the background, downloadable when ready.

♻️

Resubmission Workflow

Denied and short-paid claims flow into a dedicated resubmission queue, so recoverable revenue gets actioned instead of forgotten.

🛡️

Admin, Audit & Validation

Role-based access, user lifecycle controls, portal data validation, email alerting, and database tooling — hardened with strict CSP and security headers.

Under the hood

A modular pipeline, engineered to become multi-tenant SaaS.

The platform is already structured in clean layers — ingestion, processing, intelligence and delivery — registered as composable modules. That same shape maps directly onto a multi-tenant cloud deployment.

Layer 1

Data Sources

Regulator & payer portals
DHA Portal — claims & remittance transactions Riayati / RHA Portal — national platform sync XML files — submissions & remittance advice Facility credentials — vaulted per facility
▼
Layer 2

Sync & Ingestion Engine

Hangfire background jobs · resumable downloads
Nightly recurring sync — 90-day window, 2 AM cron Pending-download service — hosted background worker Stall detection & resume — checkpointed progress Live sync state — real-time progress to the UI
▼
Layer 3

Core Platform

ASP.NET Core · modular service registration
EF Core data layer — SQLite today, SQL Server/Postgres-ready ASP.NET Identity — roles, sessions, active-user enforcement Module system — services composed via a single registration point Security middleware — CSP, HSTS, frame/robots hardening
▼
Layer 4

Intelligence Services

Where raw files become revenue insight
XML Parsing Service Remittance Parser Reconciliation Service Report Generation — stateful, background Dashboard Service — facility health & KPIs Email Service — alerts & deliveries
▼
Layer 5

Delivery & Operations

Any device, any deployment model
Responsive web app — glassy navy/teal UI Mobile app — React Native companion Docker — volume-backed data, downloads & reports Cloud PaaS — Railway / Render manifests included On-prem service — Windows service + Cloudflare tunnel

Because background jobs, recurring syncs and hosted workers are all feature-flagged per environment, the same codebase runs as a single-clinic on-prem install or as a horizontally-scaled cloud deployment — the key enabler for the SaaS model below.

SaaS Model Projection

From proven deployment to multi-tenant platform.

Ghaf BIX is live today as a dedicated deployment. The SaaS projection is a three-phase evolution that reuses the existing modular architecture rather than rewriting it.

Phase 1

Dedicated Instances

Now → +6 months
  • One isolated instance per customer group (Docker / on-prem)
  • 13 facilities live on a single deployment
  • White-glove onboarding & credential setup
  • Validates pricing, support load & sync reliability
Phase 2

Managed Cloud

Months 6 → 18
  • Hosted single-tenant pods on Railway/Render-class infra
  • Postgres/SQL Server migration of the EF Core layer
  • Centralized billing, monitoring & automatic updates
  • Self-service facility & credential onboarding
Phase 3

Multi-Tenant SaaS

Months 18 → 36
  • Tenant-scoped data layer & shared sync worker fleet
  • Usage-metered pricing on records & facilities
  • Open API + payer/HIS integrations marketplace
  • Expansion beyond UAE: GCC regulator portals
